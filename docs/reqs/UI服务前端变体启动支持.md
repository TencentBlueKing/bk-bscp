## 需求背景

在构建产物已同时嵌入新旧两套前端的基础上（前置子需求），`bk-bscp-ui` 服务需支持按"前端变体（新/旧）"启动，运行时选择使用哪套静态资源并渲染对应 `index.html`。这样同一镜像即可通过不同启动参数分别对外提供新 UI 或旧 UI，供上层按域名分流。

## 用户故事

我想要用同一镜像通过启动参数选择渲染新 UI 或旧 UI
以便于不构建多份镜像即可分别部署两套 UI 服务

### 详细功能描述

#### [F-003] UI 服务按变体渲染

- **输入**：UI 服务启动参数（前端变体：新 / 旧）+ 运行时配置（`route_prefix`、API 地址等）
- **处理逻辑**：
  1. 服务启动时根据变体参数确定使用 `ui/dist` 还是 `ui_old/dist`
  2. 沿用现有 `index.html` 运行时注入（`BK_STATIC_URL` / `API_URL` / `SITE_URL` 等）
  3. 未指定变体时默认使用新 UI（向后兼容现状）
- **输出**：对应变体的 UI 页面
- **边界条件**：两套 UI 共用同一后端接口地址；静态资源路由（`/web/*`）与变体一致
- **异常处理**：变体参数非法时服务启动失败并给出明确提示

## 验收标准

- [ ] **AC-001**：Given 二进制含两套前端 When 以"旧 UI 变体"参数启动服务 Then 页面为旧版 UI（不含项目/环境入口）
- [ ] **AC-002**：Given 同一镜像 When 以"新 UI 变体"参数启动服务 Then 页面为新版 UI（含项目/环境管理入口）
- [ ] **AC-003**：Given 未指定变体参数 When 启动服务 Then 默认渲染新 UI（向后兼容）
- [ ] **AC-004**：Given 任一变体启动 When 访问静态资源与 SPA 路由 Then 静态资源与 index 注入均对应该变体、无 404
